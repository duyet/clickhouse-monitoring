# chmonitor docs (`apps/docs`)

Documentation site for **docs.chmonitor.dev**, built with [Astro](https://astro.build)
(static output) using a **custom theme ported from
[jordienr/astro-design-system](https://github.com/jordienr/astro-design-system)**.
Deployed to Cloudflare Workers as `chmonitor-docs`.

The site is fully self-owned (no Starlight): routing, sidebar, on-page TOC,
search ([Pagefind](https://pagefind.app)), light/dark theme, and **per-release
versioning** are all implemented in this app.

## Standalone install (important)

Like `apps/landing`, this app is **intentionally NOT a member of the root bun
workspace** (the root `package.json` `workspaces` list only names
`apps/dashboard` and `apps/mcp`). The reason: Astro requires `zod@^3`, but the
root workspace pins `zod@^4` via a load-bearing `overrides` entry that the
dashboard and AI agent depend on. Bun applies root `overrides` to every
workspace member with no way to scope an exception, which would force `zod@4`
onto Astro and break its config validation.

Keeping `apps/docs` out of the workspace gives it an isolated install where
Astro resolves its native `zod@3`. It has its own `bun.lock`.

## Develop

```bash
cd apps/docs
bun install
bun run dev      # http://localhost:4321  (search is inert in dev; built at deploy)
bun run build    # -> dist/ (runs sync-docs, astro build, then pagefind)
```

## Content & versioning

```text
docs/content/**            current / working docs (single source of truth, edited by hand)
docs/versions/<vX.Y>/**    frozen, COMMITTED per-release snapshots
apps/docs/src/content/docs/ generated by sync-docs.mjs (gitignored)
```

- **Edit docs under `docs/content/**`** — the dashboard's self-hosted `/docs`
  route reads from there too. Never edit `src/content/docs/**` (regenerated).
- `scripts/sync-docs.mjs` (prebuild) reads every `docs/versions/<vX.Y>/`,
  converts the markdown into the Astro content collection, rewrites `/docs/…`
  links to the owning version's route base, and emits the version manifest
  (`src/generated/versions.json`) and per-version sidebar
  (`src/generated/nav.json`).

### Routing

The **newest** version ("latest") is served at the site root with no prefix
(`/getting-started`, `/deploy/docker`, …). Older versions are served under
`/<version>/…`. There are **no redirects** — bare paths resolve directly to the
latest version's files, so links like `docs.chmonitor.dev/getting-started` work
as-is. A header dropdown switches versions.

### Cutting a new release

When you release, freeze the current docs into a new immutable snapshot and
commit it:

```bash
cd apps/docs
bun run snapshot v0.4        # copies docs/content -> docs/versions/v0.4
git -C ../.. add docs/versions/v0.4
```

The next build picks it up automatically, makes `v0.4` the latest, and keeps
`v0.3` available in the switcher. Versioning started at **v0.3**.

## Theme & customization

The design follows the [Vercel Docs](https://vercel.com/docs) / Geist layout
(three-column sticky shell, Geist Sans + Mono, `--ds-*` tokens, Pagefind
search). Tokens live in `design-system/docs-tokens.css` and are applied
via plain CSS — no Tailwind dependency:

- **Tokens & layout** — `src/styles/global.css`, with light + dark themes driven
  by CSS custom properties and a `[data-theme]` attribute.
- **Components** — `src/components/` (Header, Sidebar, TableOfContents,
  VersionSwitcher, ThemeToggle, Search) + `src/layouts/MainLayout.astro`.
- **Site config** — `src/config.ts` (title, links). Sidebar order/labels are
  curated in `scripts/sync-docs.mjs` (`SECTION_ORDER`, `LABEL_OVERRIDES`).
- **Logo & favicon** — `src/assets/logo.svg` and `public/favicon.svg`.
- **Fonts** — Geist Sans + Geist Mono, self-hosted via `@fontsource*`.

## Deploy

Served as static assets by a Cloudflare Worker (`wrangler.toml`) on
`docs.chmonitor.dev`. The build output includes the Pagefind search index
(`dist/pagefind/`).

**Architecture constraints (do not regress):**

- `output: 'static'` in `astro.config.mjs` — every page is HTML at build time.
- No Worker `main` script — `wrangler.toml` only declares `[assets] directory = "./dist"`.
- `public/_headers` sets long-lived `Cache-Control` on fingerprinted `/_astro/*`
  and Pagefind blobs; HTML keeps default revalidation so deploys propagate.
- `html_handling = "drop-trailing-slash"` avoids 307 redirects on bare paths.
