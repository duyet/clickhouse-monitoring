# chmonitor docs (`apps/docs`)

Documentation site for **docs.chmonitor.dev**, built with
[Astro](https://astro.build) + [Starlight](https://starlight.astro.build)
(static output). Deployed to Cloudflare Workers as `chmonitor-docs`.

Starlight owns the theme — layout, sidebar, on-page TOC, search
([Pagefind](https://pagefind.app), bundled and indexed automatically at build),
light/dark theme, and edit links. The only custom piece is a `PageTitle`
override that adds **Copy page** / **Open as Markdown** actions.

## Standalone install (important)

Like `apps/landing`, this app is **intentionally NOT a member of the root bun
workspace** (the root `package.json` `workspaces` list only names
`apps/dashboard` and `apps/mcp`). The reason: Astro requires `zod@^3`, but the
root workspace pins `zod@^4` via a load-bearing `overrides` entry that the
dashboard and AI agent depend on. Bun applies root `overrides` to every
workspace member with no way to scope an exception, which would force `zod@4`
onto Astro and break its config validation.

Keeping `apps/docs` out of the workspace gives it an isolated install where
Astro resolves its native `zod@3`. It has its own `bun.lock`.

## Develop

```bash
cd apps/docs
bun install
bun run dev      # http://localhost:4321  (search is inert in dev; built at deploy)
bun run build    # -> dist/ (runs sync-docs, then astro build incl. Pagefind)
```

## Content

```text
docs/content/**             current docs — the single source of truth, edited by hand
apps/docs/src/content/docs/ generated by sync-docs.mjs (gitignored)
```

- **Edit docs under `docs/content/**`.** Never edit `src/content/docs/**` — it is
  regenerated on every build and gitignored.
- `scripts/sync-docs.mjs` (prebuild) transforms `docs/content/**` into the
  Starlight content collection: strips legacy `nextra`/`lucide-react` imports,
  turns the first `# Heading` into the frontmatter `title`, rewrites `/docs/…`
  links to site-root paths, rewrites relative image paths to
  `raw.githubusercontent.com` URLs, and emits a per-page `editUrl` pointing at
  the real `.mdx` source on GitHub.
- A top-level `<section>.mdx` that also has a `<section>/` directory (e.g.
  `deploy.mdx` + `deploy/`) is written as `<section>/index.md` so Starlight's
  `autogenerate` sidebar lists it (labelled **Overview**, shown first).

There is **no per-release versioning** — the working docs are served directly at
the site root.

### Raw markdown

Every page has a raw-markdown sibling: `/deploy/docker` → `/deploy/docker.md`
(the root index is `/index.md`), served as `text/markdown`. It is prerendered by
`src/pages/[...slug].md.ts` and backs the **Copy page** / **Open as Markdown**
actions.

## Customization

- **Theme & sidebar** — `astro.config.mjs` (Starlight integration: title, logo,
  social links, curated sidebar order).
- **Page actions** — `src/components/PageTitle.astro` (Starlight `PageTitle`
  override).
- **Content collection** — `src/content.config.ts` (`docsLoader()` +
  `docsSchema()`).
- **Logo & favicon** — `public/favicon.svg` (+ PNG sizes, `public/og/og.png`).

## Deploy

Served as static assets by a Cloudflare Worker (`wrangler.toml`) on
`docs.chmonitor.dev`. The build output includes the Pagefind search index
(`dist/pagefind/`).

**Architecture constraints (do not regress):**

- `output: 'static'` in `astro.config.mjs` — every page is HTML at build time.
- No Worker `main` script — `wrangler.toml` only declares `[assets] directory = "./dist"`.
- `public/_headers` sets long-lived `Cache-Control` on fingerprinted `/_astro/*`
  and Pagefind blobs; HTML keeps default revalidation so deploys propagate.
- `html_handling = "drop-trailing-slash"` avoids 307 redirects on bare paths.
